## Architecture

Feature-based structure with TanStack Router file-based routing.

### Directory structure:

```
src/
  api/               # API client and endpoint functions
    client.ts        # Fetch wrapper (credentials: include, JSON, error handling)
    auth.api.ts      # signUp, signIn, getSession, signOut
  components/
    ui/              # shadcn/ui components (Button, Input, Card, Label, Form)
    layout/          # Layout components
    shared/          # Reusable shared components
  features/          # Feature modules
    auth/
      components/    # LoginForm, RegisterForm
      hooks/         # Feature-specific hooks
      schemas/       # Zod validation schemas (loginSchema, registerSchema)
  hooks/             # Global custom hooks
    use-auth.ts      # Auth hook wrapping Jotai atoms + API calls
  lib/
    utils.ts         # cn() utility (clsx + tailwind-merge)
  routes/            # TanStack Router file-based routes
    __root.tsx       # Root route (Outlet only)
    index.tsx        # Home page with redirect to login or dashboard
    auth.tsx         # Auth layout (centered card)
    auth/login.tsx   # Login page
    auth/register.tsx# Register page
    dashboard.tsx    # Protected dashboard layout (checks session)
    dashboard/index.tsx # Dashboard home
  stores/            # Jotai atoms
    auth.atom.ts     # userAtom, sessionAtom, isAuthenticatedAtom
  types/             # Shared TypeScript types
  globals.css        # Tailwind v4 directives + @theme (shadcn design tokens)
  App.tsx            # JotaiProvider + RouterProvider
  main.tsx           # Entry point with createRoot
  routeTree.gen.ts   # Generated route tree (auto-generated by Vite plugin)
```

### Key conventions:

- **Path aliases**: Use `@/` to reference `src/`. Example: `import { cn } from "@/lib/utils"`
- **Tailwind v4**: Configuration is in `globals.css` via `@theme` block. No `tailwind.config.ts` needed. Uses `@tailwindcss/vite` plugin.
- **shadcn/ui**: Components live in `src/components/ui/`. Use `components.json` for CLI config. Never modify shadcn components directly — extend via className props.
- **TanStack Router**: File-based routing. Each route file exports `Route` via `createFileRoute("/path")()`. The `routeTree.gen.ts` is auto-generated by `@tanstack/router-plugin`. Never manually edit it — the Vite plugin regenerates it on file changes.
- **Forms**: Use `react-hook-form` with `@hookform/resolvers/zod`. Form schemas go in `features/<name>/schemas/`. Use shadcn Form components (`Form`, `FormField`, `FormItem`, `FormLabel`, `FormControl`, `FormMessage`).
- **State management**: Jotai atoms in `src/stores/`. For auth, use `useAuth()` hook from `src/hooks/use-auth.ts` — it wraps the atoms and provides `login`, `register`, `logout`, `isLoading`, `isAuthenticated`.
- **API calls**: Use `apiClient` from `@/api/client` for custom endpoints. Use functions from `@/api/auth.api` for auth operations. The client automatically includes credentials and handles errors.
- **Protected routes**: Protected layout routes (like `dashboard.tsx`) use `beforeLoad` with a fetch to `/api/v1/auth/session`. Redirect to `/auth/login` if not authenticated.
- **Auth flow**: Better Auth client (`better-auth` package) is included for client-side utilities. Auth state is managed through Jotai atoms populated via the `useAuth` hook which calls `getSession()` on mount.

## Tech Stack

| Layer | Technology |
|---|---|
| Runtime | Bun |
| Framework | React 19 |
| Build Tool | Vite 8 |
| Language | TypeScript 6 |
| Routing | TanStack Router v1 (file-based) |
| Data Fetching | TanStack Query v5 |
| Tables | TanStack Table v8 |
| Styling | Tailwind CSS v4 |
| UI Components | shadcn/ui (Radix primitives) |
| Forms | React Hook Form v7 + Zod v4 |
| State | Jotai v2 |
| Dates | date-fns v4 |
| Icons | Lucide React |
| Auth | Better Auth v1 (client) |
