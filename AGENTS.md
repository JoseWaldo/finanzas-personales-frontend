## Architecture

Feature-based structure with TanStack Router file-based routing.

### Directory structure:

```
src/
  api/               # API client for non-auth endpoints
    client.ts        # Fetch wrapper (credentials: include, JSON, error handling)
  components/
    ui/              # Custom UI components (Button, Input, Card, Label, Form)
    layout/
      sidebar.tsx    # Collapsible sidebar with navigation, theme toggle, logout
    shared/
      app-logo.tsx   # Balanz brand logo (reusable, with/without text)
      theme-provider.tsx  # Syncs theme atom with HTML .dark class
      theme-toggle.tsx    # Sun/Moon toggle button
  features/          # Feature modules
    auth/
      components/    # LoginForm, RegisterForm
      hooks/         # Feature-specific hooks
      schemas/       # Zod validation schemas (loginSchema, registerSchema)
  hooks/             # Global custom hooks
    use-auth.ts      # Auth hook wrapping Better Auth client
  lib/
    auth-client.ts   # Better Auth client instance (createAuthClient)
    utils.ts         # cn() utility (clsx + tailwind-merge)
  routes/            # TanStack Router file-based routes
    __root.tsx       # Root route (Outlet only)
    index.tsx        # Landing page (hero + features + CTAs)
    auth.tsx         # Auth layout (header with logo + centered content)
    auth/login.tsx   # Login page
    auth/register.tsx# Register page
    dashboard.tsx    # Protected dashboard layout with collapsible sidebar
    dashboard/index.tsx     # Dashboard home (stats cards + empty states)
    dashboard/ingresos.tsx  # Ingresos placeholder
    dashboard/gastos.tsx    # Gastos placeholder
    dashboard/presupuestos.tsx # Presupuestos placeholder
  stores/            # Jotai atoms
    theme.atom.ts    # Theme atom (light/dark) with localStorage persistence
  types/             # Shared TypeScript types
  globals.css        # Tailwind v4 directives + @theme (design tokens)
  App.tsx            # JotaiProvider + QueryClientProvider + RouterProvider
  main.tsx           # Entry point with createRoot
  routeTree.gen.ts   # Generated route tree (auto-generated by Vite plugin)
```

### Key conventions:

- **Path aliases**: Use `@/` to reference `src/`. Example: `import { cn } from "@/lib/utils"`
- **Mobile-first**: Always write styles for mobile devices first, then use breakpoint prefixes (`sm:`, `md:`, `lg:`, `xl:`) to enhance the design for larger screens. The sidebar is an overlay/drawer on mobile (hidden by default) and inline on desktop (`md:` breakpoint).
- **Tailwind v4**: Configuration is in `globals.css` via `@theme` block. No `tailwind.config.ts` needed. Uses `@tailwindcss/vite` plugin.
- **UI Components**: Our own components in `src/components/ui/` written with Tailwind + `class-variance-authority` for variants. No Radix or shadcn dependencies.
- **TanStack Router**: File-based routing. Each route file exports `Route` via `createFileRoute("/path")()`. The `routeTree.gen.ts` is auto-generated by `@tanstack/router-plugin`. Never manually edit it — the Vite plugin regenerates it on file changes.
- **Forms**: Use `react-hook-form` with `@hookform/resolvers/zod`. Form schemas go in `features/<name>/schemas/`. Use Form components (`Form`, `FormField`, `FormItem`, `FormLabel`, `FormControl`, `FormMessage`).
- **State management**: Jotai atoms in `src/stores/` for theme only. Auth state is managed by Better Auth's reactive client (`useSession` hook).
- **API calls**: Use `apiClient` from `@/api/client` for non-auth custom endpoints. For auth operations, use the Better Auth client from `@/lib/auth-client`.
- **Protected routes**: Protected layout routes (like `dashboard.tsx`) use `beforeLoad` with `authClient.getSession()`. Redirect to `/auth/login` if not authenticated.
- **Auth flow**: Better Auth client (`better-auth/react`) initialized in `@/lib/auth-client`. The `useAuth` hook wraps `useSession()` and provides `login`, `register`, `logout`, `isLoading`, `isAuthenticated`. Session state is reactive via Better Auth's nanostores.
- **Theme**: Light/dark mode managed via `themeAtom` (Jotai + localStorage). `ThemeProvider` syncs the `dark` class on `<html>`. An inline `<script>` in `index.html` applies the class before React loads to prevent flash. `ThemeToggle` provides a sun/moon button. If no preference is stored, system `prefers-color-scheme` is used as default.

For visual and brand guidance, see [DESIGN.md](DESIGN.md).

## Tech Stack

| Layer | Technology |
|---|---|
| Runtime | Bun |
| Framework | React 19 |
| Build Tool | Vite 8 |
| Language | TypeScript 6 |
| Routing | TanStack Router v1 (file-based) |
| Data Fetching | TanStack Query v5 |
| Tables | TanStack Table v8 |
| Styling | Tailwind CSS v4 |
| UI Components | Custom (class-variance-authority + Tailwind) |
| Forms | React Hook Form v7 + Zod v4 |
| State | Jotai v2 (theme), Better Auth (auth) |

| Icons | Lucide React |
| Auth | Better Auth v1 (client + server) |
